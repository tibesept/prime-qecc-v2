# Prime Number QECC

This is a numerical toy model in Python that attempts to build a
computational bridge between three otherwise unconnected mathematical
frameworks:

- **Weil's explicit formula** (analytic number theory): a distribution
  that relates the prime numbers and the non-trivial zeros of the
  Riemann zeta function. Weil positivity of this distribution is
  known to be equivalent to the Riemann Hypothesis.

- **Bost-Connes system** (noncommutative geometry): a quantum
  statistical mechanical system whose Hamiltonian is built from
  logarithms of primes, whose partition function is exactly the
  Riemann zeta function, and which undergoes a phase transition
  precisely at the pole Î² = 1.

- **p-adic AdS/CFT** (Gubser et al., arXiv:1605.01078): a holographic
  model in which the bulk of spacetime is replaced by a Bruhat-Tits
  tree - a discrete fractal graph generated by a prime number p.

## Hypothesis

Holographic spacetime in standard AdS/CFT is known to function as a
quantum error-correcting code (Almheiri, Dong, Harlow, arXiv:1411.7041).
If we build holographic spacetime out of prime-number fractals (p-adic
AdS/CFT), does the thermodynamic stability of a prime-based system
(Bost-Connes) correspond to the error-correcting stability of that
spacetime?

Concretely: when a Riemann zero is perturbed off the critical line,
the Weil functional W(f) loses positivity. This project numerically
tracks whether that loss of positivity propagates as a breakdown of
unitarity in the edge weights of a Bruhat-Tits tree â€” a proxy for a
p-adic holographic tensor network.

This is not a proof of anything. It is an attempt to see if the
analogy holds up computationally before asking whether it can be
made rigorous.

## Project Structure

- `data/` : Caches for Riemann zeros data.
- `src/` : Python source files for the implementation.
  - `data_loader.py`: Fetches dataset from Odlyzko tables.
  - `weil_*.py`: Computes terms for the Weil explicit functional formula.
  - `bruhat_tits.py`: Constructs the p-adic Bruhat-Tits tree.
  - `connection.py`: Connects structural properties evaluated.
  - `dashboard.py`: Renders Plotly outputs.
- `tests/`: Basic validation scripts.
- `requirements.txt`: Python package requirements.

## Running

1. `python3 -m venv venv`
2. `source venv/bin/activate`
3. `pip install -r requirements.txt`
4. `cd src && python main.py`

## Visualizations

The model produces HTML visualization dashboards demonstrating the mathematical concepts studied. Open `dashboard.html`, `weil_components.html`, and `tree_initial.html` in your browser.
